Brian

Easy communication and media access for people with cognitive disabilities

Discover Brian and collaborate

Features

Brian is a modular platform that supports easy, uncomplicated media sharing and communication between people. It features a non intimidating user experience designed for people with cognitive disabilities. A control panel is provided to manage the end user experience which simplify access to existing on-line services for email and media. Brian acts as a bridge to the easier end user experience.

For example, Brian includes an email client with minimal features and an uncluttered user interface. This is controlled by the control panel which allows the selection of an email service and other options. In addition to email, there is a range of media players for exploring photos and videos selected from on-line collections.

Here are mockups of the email viewer and config screens and full mockups are available for the user and carer (pdfs).

While brian is designed primarily for users with cognitive disabilities who dislike complexity and distractions it may also be of use for people with very low digital literacy.

Key features

  • Enables communication and sharing between end users and carers both local and remote.
  • Different UX for user and carer but both are uncluttered.
  • Friendly and simplified UI for end user to accesses communications and media.
  • Several levels of complexity available for end user UX to match preferences to provide stepping stones.
  • Use of existing services by carer e.g. Gmail, Picassa and WebRTC.
  • Management of the end user experience by carers using a control panel.
  • Features include email, video calls, photo viewer, video viewer.

Project Goals

As an open source project Brian aims to be a platform supporting concrete services. It is not quite an end user ready product itself. It also aims to be a social 'hub' for developers, academics, service providers and users interested in exploring solutions to cognative accessibility.

Two services based on Brian are currently planned:

  • EasyOne: for family members wanting to stay in contact with and support remote elderly relatives with early stage dementia.
  • CIRCA: a tool to encourage memory based discourse between people with dementia and carers in a residential setting.

Project provenance and collaborations

Brian directly builds on the experience gained with the requirments of people with dementia through several projects:

  • Maavis provides structured media and communications acccess for people with dementia.
  • EasyOne a prototype of a system for access to email and photos by elders with low digital literacy.
  • CIRCA an easy-to-use computer programme that helps support conversation between people with dementia and their caregivers or relatives.
  • W3C cognitive accessibility task force exploring web accessibility requirements.

Brian is part of the the Prosperity4All European Union funded project that is devleoping key parts of the GPII global public infrastructure for accessibility and assistive technology. In this context Brian is contributing components to the Developers Space code repository. It is also including several other modules from the DSPspace to enhance its feature set.

Several of the partners in the InLife European project are collaborating with OpenDirective, bringing their extensive user requirements experience to Brian. In particular, features required for a HTML realisation of the CIRCA project will be added to Brian.

Technical features

  • Optimised for Desktop and tablet form factors.
  • Latest HTML web technology including node.js server.
  • Works with existing services such as GMail and Picassa.
